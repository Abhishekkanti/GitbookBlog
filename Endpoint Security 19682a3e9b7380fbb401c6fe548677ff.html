Endpoint Security

What is Endpoint?

  • Endpoint is any physical hardware device that is connected over a network
  • It might wide range of devices like desktops, laptops, mobile devices, server(web, email, DB), IOT(printer, camera), routers, switches, firewalls, etc.

Endpoint Security Controls

  1. Antivirus / Antimalware
    • Scans files and activities
    • Matching patterns and signatures
  1. Endpoint Detection and Response (EDR)
    • Real-time monitoring and response
    • Agent-based deployment
    • Monitor process, file, registry, and network activity
  1. Extended Detection and Response (XDR)
    • Integration of multiple security controls and telemetry
    • Runbooks and automated response to routine threats
  1. Data Loss Prevention (DLP)
    • Protect sensitive data at rest, transit, and in processing
    • Access controls, data masking, prevention
  1. User and Entity Behavior Analytics (UBA)
    1. Monitoring user behavior patterns
    1. Detect deviations from historic and contextual baseline
    1. Insider threats, account compromise, data exfiltration
  1. HIDS / HIPS
    1. Host-based Intrusion Detection System (HIDS)
    1. Host-based Intrusion Prevention System (HIPS)
  1. Host-based Firewall
    1. Controls incoming and outgoing traffic on a host

Endpoint Security Monitoring

  1. Process Execution
    1. Monitoring running processes
    1. Executable files, PIDs, command-line arguments
    1. Parent-child process hierarchy (rats and worms)
  1. File System Changes
    1. Creation, modification, deletion
    1. File Integrity Monitoring (FIM) (checks against initial checksum of file)
  1. Network Connections
    1. Traffic and connections initiated from the endpoint
    1. Associated processes and executables.
  1. Registry Modifications
    1. Monitor registry keys and values
    1. Detect backdoors, persistence, detection evasion. (disabling AV or EDR)

Processes Analysis

1. Windows Process Analysis

  1. Running instances of programs and applications

Windows Core Processes

  1. System Process
    1. System process is the main process it is responsible for kernel mode system thread (interacting with hardware), managing cpu, memory, disk, drivers, and core operations.
    1. It always has the PID of 4.
    1. If the System is not running with PID=4 then it is suspicious.
  1. smss.exe (Session manager subsystem)
    1. Windows Session Manager
    1. Initiating and managing user sessions
    1. Launches child processes - wininit.exe, csrss.exe
  • If more than >1 smss processes are running or path different or parent process then it is suspicious.

  1. csrr.exe (client/server runtime subsystem)
    1. responsible for managing console windows
    1. Importing DLLs for the Windows API
    1. GUI tasks around shutdown.

Each user has his own session separated for others for data and security purpose.

  1. wininit.exe (Windows Initialization)
    1. Initialize all the things! (helps in booting)
    1. Session 0
    1. Spawns child processes (services.exe, Isass.exe)

  1. services.exe (Service Control Manager)
    1. Service Control Manager (SCM)
    1. Starting, stopping, and interacting with services
    1. Sets the LastKnownGood CurrentControlSet registry value

  1. svchost.exe (Service Host)
    1. Hosting and managing Windows services
    1. Used to run service DLLs
    1. Runs with the -k parameter to differentiate instances/services in specific group.

  1. Isass.exe (Local Security Authority Subsystem Service)
    1. Authenticating users
    1. Implementing local security policies
    1. Writing events to the security event log

  1. winlogon.exe (Windows Logon)
    1. Manages login and logout procedures
    1. Loads user profiles (NTUSER.DAT)
    1. Responds to the Secure Attention Sequence (SAS)

  1. explorer.exe (Windows Explorer)
    1. Provides the GUI for files, folders, and system settings
    1. Manages the taskbar, Start Menu, and desktop
    1. Responsible for the overall desktop environment

Sysmon

"Sysmon," short for "System Monitor." Sysmon is a tool within the Sysinternals Suite by Microsoft, used to monitor and log system activity into the Windows Event Log. This can be incredibly useful for tracking and analyzing potential security incidents.

Sysmon records detailed information about process creations, network connections, file creation time changes, and more. Here's a brief overview of some of its main features:

  1. Process Creation Monitoring: Logs details about process creations, including process IDs, parent process IDs, and command-line arguments.
  1. Network Connection Monitoring: Captures information about network connections, such as source and destination IP addresses, ports, and protocols.
  1. File Creation Monitoring: Tracks file creation and file modification events.

Sysmon Lets you to filter log based on their type given below. Sysmon generates a variety of event IDs, each corresponding to a specific type of system activity. Here are some of the key event IDs and what they represent:

  • Event ID 1: Process creation (logs process creation with command line and hash).
  • Event ID 2: File creation time changed (logs when a file creation time is changed).
  • Event ID 3: Network connection (logs TCP/UDP connections).
  • Event ID 4: Sysmon service state changed (logs Sysmon service state changes).
  • Event ID 5: Process terminated (logs when a process terminates).
  • Event ID 6: Driver loaded (logs when a driver is loaded).
  • Event ID 7: Image loaded (logs when an image is loaded).
  • Event ID 8: CreateRemoteThread (logs when a thread is created in another process).
  • Event ID 9: RawAccessRead (logs when a process conducts reading of raw data from the disk).
  • Event ID 10: ProcessAccess (logs when one process opens another process).
  • Event ID 11: FileCreate (logs when a file is created or overwritten).
  • Event ID 12: RegistryEvent (Object create and delete - logs when registry keys and values are added/deleted).
  • Event ID 13: RegistryEvent (Value Set - logs registry value modifications).
  • Event ID 14: RegistryEvent (Key and Value Rename - logs registry key and value renames).
  • Event ID 15 : FileCreateStreamHash (logs downloads and its hash)
  • Event ID 22 : DNSEvent (DNS Query) (Log whenever process executes DNS query)
  • Event ID 29 : FileExecutableDetected (Log creation of new executable) Examines file by checking file beginning for file signature.

A process can still be running in the system even after the source file is deleted. This is because the file is stored in memory while it is executing. (like if delete the payload while it is running the payload is still active until its process is killed).

Scheduled Tasks in Linux

Cron :

Cron is a powerful tool for automating file management, process execution, and task scheduling in Linux systems. By leveraging Cron, you can efficiently manage repetitive tasks, automate workflows, and improve system efficiency.

Cron allows users to schedule tasks, often referred to as Cron jobs, to run periodically at fixed times, or dates or intervals.

It is widely used to automate tasks like like running backups or system maintenance or regular data processing jobs.

Each user has his own Cron job files, and a system wide Cron job files for system

Format:

>> 05 00 * * * /tmp/backup.sh
  • The first 2 places represents interval in minutes (here, execute after interval of 5 min for given hour, */5 = to execute every 5th minute),
  • The next 2 places represents hour (time) of day to execute (here every day at 12:00),
  • The next “* * * “represents day/month/day of week for execution, you can use month number or short form Jan, Feb, etc / for day of week use week number of short name (sun, mon or 0=7= sun, 1 = mon,…) here, its execute everyday.
  • command field is used to execute specific file.

Use corntab.guru site to create Cron jobs :

Command “etc/crontab” is used to display all the Cron job entries files. As you can see there are some pre defined Cron jobs like daily, weekly, monthly that gets executed.

You can open Cron jobs files for further investigate (like extracting hashes for each files).

  1. >> ls -al /var/spool/cron/crontabs : To list out all the Cron tabs files of the system (all user levels).
  1. >> crontab -e [1-3] : To create your own Cron job for user level jobs (run this command & select editor).

    Eg : */5 * * * * ~/home/kali/Download/evil.elf

  1. >> sudo crontab -u user : To display all the Cron jobs

Traverse all the Cron files to find any malicious Cron jobs.
